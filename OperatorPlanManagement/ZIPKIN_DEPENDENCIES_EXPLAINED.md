## 1. **spring-cloud-starter-sleuth**

### What it is:
Spring Cloud Sleuth is the entry point for Spring Cloud's distributed tracing solution. It automatically instruments your Spring Boot application to generate trace IDs and span IDs for every request.

### Why you need it:
- Automatically generates unique trace IDs for request chains
- Automatically propagates trace IDs across services
- Integrates with Logback to add trace information to logs
- Zero configuration needed for basic functionality

### What it does:
```
Request Flow:
Client → API Gateway → Service1 → Service2 → Database
[trace-id: abc123]   [trace-id: abc123]  [trace-id: abc123]
All operations are tagged with the same trace ID
```

### Dependency Details:
```xml
<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-sleuth</artifactId>
</dependency>
```

- **Group:** `org.springframework.cloud`
- **Artifact:** `spring-cloud-starter-sleuth`
- **Version:** Managed by Spring Cloud BOM (2025.0.1)
- **Scope:** Compile (default)

## 2. **micrometer-tracing-bridge-brave**

### What it is:
Micrometer is a metrics and tracing facade that provides a standard way to collect observability data. This artifact is the bridge that connects Micrometer's tracing APIs to Brave (the actual tracing implementation).

### Why you need it:
- Provides the abstraction layer for tracing
- Allows you to switch tracing implementations without code changes
- Integrates with Spring Boot's Micrometer framework
- Enables auto-configuration of tracing

### What it does:
```
Spring Boot Application
        ↓
Micrometer Tracing API (abstraction)
        ↓
Brave Bridge (this dependency)
        ↓
Brave Implementation (actual tracing)
        ↓
Zipkin (trace storage/visualization)
```

### Dependency Details:
```xml
<dependency>
    <groupId>io.micrometer</groupId>
    <artifactId>micrometer-tracing-bridge-brave</artifactId>
</dependency>
```

- **Group:** `io.micrometer`
- **Artifact:** `micrometer-tracing-bridge-brave`
- **Version:** Managed by Spring Cloud BOM
- **Scope:** Compile (default)
- **Replaces:** Old `spring-cloud-sleuth-zipkin` approach

---

## 3. **brave-core**

### What it is:
Brave is a low-level distributed tracing library created by Zipkin's team. It handles the actual instrumentation of HTTP requests, database calls, and other operations.

### Why you need it:
- Provides HTTP client/server instrumentation
- Instruments database connections (JDBC)
- Handles trace context propagation
- Manages span creation and lifecycle
- Lightweight and performant

### What it does:
```
When a request comes in:
1. Brave intercepts the request
2. Creates a span (named operation)
3. Assigns trace ID if new, propagates if existing
4. Collects timing information
5. Sends span to Zipkin reporter
6. Closes span when operation completes
```

### Dependency Details:
```xml
<dependency>
    <groupId>io.zipkin.brave</groupId>
    <artifactId>brave-core</artifactId>
</dependency>
```

- **Group:** `io.zipkin.brave`
- **Artifact:** `brave-core`
- **Version:** Managed by Spring Cloud BOM
- **Scope:** Compile (default)
- **Purpose:** Core tracing instrumentation

---

## 4. **zipkin-reporter-brave**

### What it is:
The Zipkin Reporter is responsible for collecting spans from Brave and sending them to the Zipkin server. This is the bridge between your application and the Zipkin backend.

### Why you need it:
- Collects traces generated by Brave
- Batches spans for efficient transmission
- Handles communication with Zipkin server
- Manages reporter lifecycle
- Provides reliability and error handling

### What it does:
```
Application Spans
        ↓
Zipkin Reporter
(collects & batches spans)
        ↓
HTTP POST to Zipkin Server
(http://zipkin-server:9411/api/v2/spans)
        ↓
Zipkin Server
(stores and indexes traces)
```

### Configuration in application.properties:
```properties
management.tracing.sampling.probability=1.0
management.zipkin.tracing.endpoint=http://localhost:9411
```

### Dependency Details:
```xml
<dependency>
    <groupId>io.zipkin.reporter2</groupId>
    <artifactId>zipkin-reporter-brave</artifactId>
</dependency>
```

- **Group:** `io.zipkin.reporter2`
- **Artifact:** `zipkin-reporter-brave`
- **Version:** Managed by Spring Cloud BOM
- **Scope:** Compile (default)

---

## 5. **zipkin-sender-okhttp3**

### What it is:
The Zipkin Sender is the HTTP transport layer that actually sends trace data to the Zipkin server. This specific implementation uses OkHttp3 (a modern HTTP client library).

### Why you need it:
- Sends traces to Zipkin server via HTTP
- Uses OkHttp3 for reliable connection management
- Handles connection pooling
- Provides automatic retries
- Supports gzip compression

### What it does:
```
Batched Spans (from zipkin-reporter-brave)
        ↓
Zipkin Sender (okhttp3)
        ↓
HTTP POST Request
POST /api/v2/spans HTTP/1.1
Host: localhost:9411
Content-Type: application/json
Content-Encoding: gzip

[span data]
        ↓
Zipkin Server Response
HTTP/1.1 202 Accepted
```

### Dependency Details:
```xml
<dependency>
    <groupId>io.zipkin.reporter2</groupId>
    <artifactId>zipkin-sender-okhttp3</artifactId>
</dependency>
```

- **Group:** `io.zipkin.reporter2`
- **Artifact:** `zipkin-sender-okhttp3`
- **Version:** Managed by Spring Cloud BOM
- **Scope:** Compile (default)
- **Why OkHttp3:** Modern, efficient, supports HTTP/2

## 🔄 How They Work Together

### Request Flow with Tracing:

```
1. HTTP Request arrives
   └─ spring-cloud-starter-sleuth intercepts it

2. Brave takes over
   └─ brave-core creates a span
   └─ assigns trace ID if new
   └─ measures execution time

3. Operations execute
   └─ Database queries
   └─ Service calls
   └─ Cache operations
   (All automatically instrumented by Brave)

4. Response sent back
   └─ Span is marked complete
   └─ Timing information collected

5. Span reporting
   └─ zipkin-reporter-brave batches the span
   └─ zipkin-sender-okhttp3 sends HTTP POST
   └─ Zipkin server receives and stores

6. Zipkin UI
   └─ Displays complete trace
   └─ Shows latency breakdown
   └─ Shows service dependencies
```

## ⚙️ Configuration Needed

Add this to your `application.properties`:

```properties
# Tracing Configuration
management.tracing.sampling.probability=1.0
management.zipkin.tracing.endpoint=http://localhost:9411

# Optional: Enable trace logging
logging.level.org.springframework.cloud.sleuth=DEBUG
logging.level.brave=DEBUG
```

### Configuration Options:

| Property | Value | Purpose |
|----------|-------|---------|
| `management.tracing.sampling.probability` | 0.0-1.0 | % of requests to trace (1.0 = 100%) |
| `management.zipkin.tracing.endpoint` | http://... | Where to send traces |
| `management.zipkin.tracing.connect-timeout` | 5000 | Connection timeout in ms |
| `management.zipkin.tracing.read-timeout` | 5000 | Read timeout in ms |
